5. BACKEND EXPLANATION — app.py
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ENGLISH:
"The app.py file is the heart of the backend. Let me explain
 each part:

 A) COMPANY DATA (dictionary)
    All company details like name, phone, address, GST number
    are stored in a Python dictionary called 'COMPANY'.
    This data is passed to templates automatically.

 B) PRODUCT CATEGORIES (list of dictionaries)
    All 9 product categories with their sub-products are
    stored in 'CATEGORIES' — a list of dictionaries.
    Each category has: id, name, url, and sub-products.

 C) CONTEXT PROCESSOR
    @app.context_processor automatically injects 'company'
    and 'categories' into every template — so we don't need
    to pass them manually in every route.

 D) ROUTES
    @app.route('/') → renders homepage (index.html)
    @app.route('/about') → renders about page
    @app.route('/products') → renders products page
    @app.route('/products/<cat_id>') → renders specific category
    @app.route('/services') → renders services page
    @app.route('/contact') → renders contact page
    @app.route('/api/enquiry', POST) → handles form submission

 E) API ENDPOINT
    The /api/enquiry route accepts POST requests (JSON data)
    from the contact form and returns a JSON response.
    In production, this would save to a database or send email."

GUJARATI:
"app.py backend ની main file છે:

 A) COMPANY dictionary — company ની details
 B) CATEGORIES list — products ની categories + sub-products
 C) Context Processor — company+categories automatically
    દરેક template ને inject કરે
 D) Routes — દરેક page ને @app.route() define કરે

ENGLISH:
"The main.js has 4 main features:

 1. NAVBAR SCROLL SHADOW
    Adds a shadow to the navbar when user scrolls down.
    Uses window.addEventListener('scroll', ...)

 2. HAMBURGER / MOBILE MENU
    Click hamburger → nav slides open on mobile.
    Spans animate into X shape.
    Click outside → menu closes.

 3. AUTO-PLAYING SLIDER
    setInterval runs every 4.5 seconds → next slide.
    prev/next buttons + dot indicators work.
    Clicking dot or arrow resets the timer.

 4. AJAX FORM SUBMISSION
    fetch('/api/enquiry', POST) sends form data to Flask.
    No page reload — success message shown inline.
    Works for homepage form, contact form, services form.

 5. SCROLL REVEAL ANIMATION
    IntersectionObserver watches product cards.
    When card enters viewport → fade in + slide up.
    Staggered delay (80ms each) for natural effect."

COMMON QUESTIONS & ANSWERS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Q: Why Flask and not Django?
A: Flask is lightweight and simple. For this size of project
   (6 pages, no complex DB), Flask is perfect. Django would
   be overkill.
   "Flask lightweight છે. 6 pages project માટે Flask ideal."

Q: Why no database?
A: For this project, all data is static (company info, product
   names). In production, we would add SQLite/PostgreSQL
   to store enquiries from the contact form.
   "Demo project — data Python dictionary માં. Production
    માં SQLite/PostgreSQL add."

Q: What is Jinja2?
A: Jinja2 is Flask's template engine. It lets us write Python
   logic inside HTML — like for loops, if conditions, and
   variable output using {{ }} and {% %} syntax.
   "Jinja2 HTML ની અંદર Python logic write કરવા દે."

Q: How is it responsive?
A: CSS media queries at 1024px, 900px, 768px, 480px change
   the grid from multi-column to single column. The navbar
   becomes a hamburger menu on mobile.
   "CSS media queries — mobile ઉપર hamburger menu."

Q: Can this be deployed online?
A: Yes. Add 'gunicorn' and deploy on Railway or Render for free.
   "gunicorn add કરો, Railway/Render ઉપર free deploy."
